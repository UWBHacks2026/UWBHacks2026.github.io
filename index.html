Skill Bridge

A skill-based job matching web application

Project Overview

Skill Bridge is a web application designed to help candidates connect their skills with real job opportunities. Candidates enter their profile information, skills, languages, location, and job preferences. The system then uses this information to search job listings from USAJobs, extract relevant job skills, compare them with the candidate’s skills, and return matched job results.

Project Purpose

The purpose of Skill Bridge is to reduce the gap between job seekers and job opportunities by focusing on skills instead of only job titles. Many candidates may have useful skills but may not know what jobs match their abilities. Skill Bridge helps candidates discover jobs that align with their existing skills.

Core Features

  • Create and store candidate user accounts.
  • Create candidate profiles with personal and location information.
  • Accept candidate skills as user input.
  • Clean and extract skills from user-provided text.
  • Store candidate skills and languages in separate database tables.
  • Search USAJobs using candidate skills and county location.
  • Parse USAJobs API results into clean job objects.
  • Save job listings from USAJobs into the database.
  • Extract job-related skills from job descriptions.
  • Compare candidate skills with job skills.
  • Return matched jobs with a match score.

Frontend Display

The frontend does not directly communicate with the database. Instead, the frontend sends requests to API routes. The API routes process the request, call helper functions, interact with the database, and return a response back to the frontend.

Database Design

The database contains tables for users, candidate profiles, candidate skills, candidate languages, jobs, and job skills.

Main Tables

  • users: Stores account-level information such as email, password hash, role, and created date.
  • candidate_profiles: Stores candidate-specific details such as name, phone, county, state, bio, availability, and preferred job type.
  • candidate_skills: Stores each candidate skill as a separate row linked to a candidate profile.
  • candidate_languages: Stores each candidate language as a separate row linked to a candidate profile.
  • jobs: Stores job listings pulled from USAJobs, including title, organization, description, location, apply URL, pay, category, and posting date.
  • job_skills: Stores extracted job skills as separate rows linked to a job.

Why Skills and Languages Are Separate Tables

Skills and languages are stored in separate tables because one candidate can have many skills and many languages. Instead of storing all skills in one text column, each skill is stored as its own row. This makes it easier to search, compare, and match candidate skills with job requirements.

API Routes

The project uses API routes to connect the frontend with the backend and database.

Users Routes

  • GET /users - Gets all users.
  • POST /users - Creates a new user.

Candidates Routes

  • GET /candidates - Gets all candidate profiles.
  • POST /candidates - Creates a candidate profile with skills and languages.
  • GET /candidates/:id/skills - Gets skills for a specific candidate.
  • GET /candidates/:id/languages - Gets languages for a specific candidate.

Jobs Routes

  • GET /jobs - Gets saved jobs from the database.
  • GET /jobs/:id/skills - Gets extracted skills for a specific job.
  • POST /jobs/search-match - Searches USAJobs, saves jobs, compares skills, and returns matched jobs.

USAJobs API Integration

The project uses the USAJobs API to search for real federal job listings. The API returns complex job data, so the backend parses the information into a cleaner job object that is easier to use in the frontend and database.

Parsed Job Object

  category: "Job category",

  color: "UI color",
  initials: "Organization initials",
  description: "Job description",
  applyUrl: "Application link"
}

Skill Extraction and Data Cleaning

Candidate skills and job skills may be entered as long text or comma-separated values. The backend uses a skill extraction function to normalize and identify skills from user input or job descriptions.

Example:

Input:
"I have experience with JavaScript, Python, customer service, and problem solving."

Output:
["Programming", "Customer Service", "Critical Thinking"]

This helps the system store skills consistently and improves the accuracy of job matching.

Job Matching Logic

The matching system compares candidate skills with extracted job skills. Each job receives a match score based on how many required job skills overlap with the candidate’s skills. 

Candidate Skills:
["Programming", "Customer Service", "Writing"]

Job Skills:
["Programming", "Writing", "Management"]

Matched Skills:
["Programming", "Writing"]

Match Score:
2 / 3 = 67%

Current User Flow

  1. Candidate enters profile information on the frontend.
  2. Frontend sends candidate data to the API.
  3. API cleans and extracts candidate skills.
  4. Candidate profile, skills, and languages are saved in the database.
  5. Candidate requests job matches.
  6. Backend searches USAJobs using candidate skills and county.
  7. USAJobs results are parsed into clean job objects.
  8. Jobs and job skills are saved into the database.
  9. System compares candidate skills with job skills.
  10. Frontend displays matched jobs with match scores.

Important Files

schema.ts

Defines the database tables and relationships using Drizzle ORM.

queries.ts

Contains reusable database functions such as creating users, creating candidates, saving jobs from USAJobs, getting skills, and matching candidates to jobs.

dataCleaning.ts

Contains helper functions for extracting and normalizing skills from raw text input.

searchFile.ts

Contains the USAJobs search logic. It searches jobs by skills and county, parses raw API data, and returns clean job listings.

routes/users.ts

Handles user account routes.

routes/candidates.ts

Handles candidate profile creation, candidate skills, and candidate languages.

routes/jobs.ts

Handles USAJobs searching, saving jobs to the database, and returning matched job results.

Summary

The application collects candidate information, cleans and extracts skills, searches for real job listings, saves job data, compares skills, and returns matched jobs with scores.
